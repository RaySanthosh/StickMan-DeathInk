### 2.3 Authentication Setup

1. Go to Firebase Console → Authentication → Sign-in method
2. Enable **Google** provider
3. Set support email: `iyappansanthosh2004@gmail.com`
4. **Add SHA-1 and SHA-256 fingerprints** for your release key:
   ```bash
   keytool -list -v -keystore android/app/death-note-release.jks -alias death-note
   ```
   Copy both `SHA1` and `SHA256` values.
5. Go to Firebase Console → Project Settings → Your Apps → Android app
6. Click "Add fingerprint" and paste both SHA values
7. **Re-download `google-services.json`** after adding fingerprints and replace `android/app/google-services.json`

> **CRITICAL:** Google Sign-In will NOT work in release builds without the release SHA fingerprints registered in Firebase.

### 2.4 Firestore Security Rules (Production)

Go to Firebase Console → Firestore → Rules and set:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Users can read/write only their own save data
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }

    // Leaderboard: anyone signed in can read, users can write their own entry
    match /leaderboard/{entryId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null && request.auth.uid == entryId;
    }

    // Death note entries: anyone signed in can read, users write their own
    match /deaths/{entryId} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
    }

    // Deny everything else
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
```

> **WARNING:** If your rules are still the default `allow read, write: if true`, change them NOW. Open rules = anyone can read/write/delete all your data.

### 2.5 Firestore Indexes

If the app queries leaderboard with `orderBy` + `where`, you may need composite indexes. Firebase will log an error URL in the console — click it to auto-create the index.

### 2.6 Firebase Cloud Messaging

1. Go to Firebase Console → Cloud Messaging
2. Ensure it's enabled (it is by default)
3. No server key needed for Flutter — `firebase_messaging` handles it via `google-services.json`
